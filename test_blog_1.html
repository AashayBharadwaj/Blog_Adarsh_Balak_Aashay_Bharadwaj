SQL

SQL is a powerful programming language for managing and manipulating data stored in relational databases. In this blog post, we'll take a closer look at some of the intermediate SQL techniques for filtering, aggregating, sorting, and grouping data.

Filtering Data
The WHERE clause in SQL allows you to filter rows from a query based on specific conditions.
For example, suppose we have a table of customer data with columns for id, name, and state. We can use the WHERE clause to retrieve only the rows for customers who live in a particular state:

Query

SELECT * FROM customers WHERE state = 'CA';

The above query will return the customers with state as CA.

The following query will return all rows from the customers table where the state column is equal to 'CA'. You can also use comparison operators such as <, >, <=, >=, and <> to specify more complex conditions. For example, the following query will return rows where the id column is greater than 10:

Query

SELECT * FROM customers
WHERE id > 10


Aggregate Functions

SQL provides a number of built-in functions for performing calculations on data.
These are known as aggregate functions, and they include functions such as SUM, AVG, MIN, and MAX.
Aggregate functions are typically used in conjunction with the GROUP BY clause to calculate values for each group.

For example, suppose we have a table of orders with columns for order_id, product, and quantity. We can use the SUM function to calculate the total quantity of each product sold:

Query

SELECT product, SUM(quantity)
FROM orders
GROUP BY product;

This query will return a list of products and the total quantity sold for each product. Other common aggregate functions include AVG (average), MIN (minimum), and MAX (maximum).
